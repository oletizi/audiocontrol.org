# Reference: editorialcontrol vs audiocontrol — how the voices differ

The two sites are siblings and share DNA: no marketing fluff, concrete receipts, typographic care, "no form, no funnel" ethic, same author. But the registers are different in ways that matter on the page.

Read this if you're familiar with audiocontrol voice and need to switch — or if you find a draft drifting into the wrong register.

## One-line summary

- **audiocontrol** is a **service manual**.
- **editorialcontrol** is a **publication**.

## Side-by-side

| Axis | audiocontrol | editorialcontrol |
| --- | --- | --- |
| Frame | Hardware + software documentation | Magazine / publication |
| Unit of publication | Blog post | Dispatch |
| Vocabulary | "Editors," "hardware," "docs," "build" | "Dispatch," "desk," "issue," "press," "editor" |
| Post index | "Latest Writing" | "The desk" |
| Newsletter | None / mailing list | "The dispatch," editor mailto |
| Typography marker | `→ ↳ · §`, Departure Mono | `◆ § No. — ·`, Fraunces + JetBrains Mono |
| Sentence length | Short default; long earns | Long OK when earned; short lands arguments |
| Structural move | Spec line, bolded lead-ins, tables | Thesis → two failure modes → third option |
| Opening line of longform | A hard-problem statement | A concession to the consensus, then a pivot |
| Closing line of longform | Compressed claim ("The cycle compounds.") | Compressed observation ("…so the content got made.") |
| Signature phrase | "How do you know that?" | "The receipts stay on the page." / "No form, no funnel." / "Published from a terminal." |
| Protagonist | The hardware | The craft, the workflow, the publication itself |
| Uses tables | Yes (metrics, specs) | Yes (same) |
| Uses code | Yes (protocol details, file paths) | Yes (same) |
| Bolded lead-in paragraphs | Yes | Yes |
| Em-dash as connector | Yes | Yes (more of it) |
| Semicolons | Rare | Rare |
| Italic quoted speech | Heavily used for agent/user quotes | Used for emphasis more than quotation |

## Common bleed scenarios

### audiocontrol bleeding into editorialcontrol

Symptoms:
- Terse, bullet-heavy drafts where prose would be more on-brand.
- Spec lines in contexts that don't need them.
- "Release notes" feel when the piece should be argumentative.
- Numbered shipping dates and bit depths appearing in a piece that's about workflow, not hardware.

Fix:
- Let sentences breathe. Add connective clauses with em-dashes.
- Replace bullet lists with prose unless the list is a genuine enumeration (skills, failure modes).
- Think "editor's column," not "changelog."

### editorialcontrol bleeding into audiocontrol

Symptoms:
- Long, literary sentences in what should be a hardware card.
- Magazine scaffolding ("In this dispatch," "Vol./Issue stamp") on an audiocontrol page.
- A "thesis → two failure modes → third option" argument in a blog post about the Roland S-550.
- Em-dash asides that soften what should be a declarative line.

Fix:
- Compress. Cut connective tissue.
- Lead with the hardware, the spec, the year-voice-bit line.
- Let short declarative sentences stand.

## When it's appropriate for them to cross-reference each other

- **editorialcontrol** routinely references audiocontrol as "the sibling site" or "the practical project where these workflows actually run." It's the source of most receipts. This is on-brand.
- **audiocontrol** can reference editorialcontrol as "the sibling publication" or "where we write about the workflow side of this work." It's fine in footers, About pages, and a tail paragraph of a longform process post. It's not appropriate in a hardware card or an editor page — those stay focused on the device.

Both footers already carry a "Sibling" block linking to the other. Don't repeat the link in body copy without a reason.

## Shared phrases and where they fit

- **"The receipts stay on the page."** — editorialcontrol. (Can appear in audiocontrol longform about process/meta, but is more at home in the publication.)
- **"How do you know that?"** — audiocontrol. (Specifically about the agent-challenge moment. Don't migrate it.)
- **"No form, no funnel."** — editorialcontrol. (About the publication's relationship to marketing instrumentation.)
- **"Published from a terminal."** — editorialcontrol. (Brand line; doesn't belong on audiocontrol.)
- **"The cycle compounds."** — audiocontrol. (Structural-improvement claim; very audiocontrol.)
- **"Open source · Web-native"** — audiocontrol. (Spec-block language.)

## Test: read aloud

The fastest way to tell if a draft is in the right voice: read it aloud.

- If it sounds like a field engineer writing up what they found on the rack, it's audiocontrol.
- If it sounds like a thoughtful operator dictating an editor's column from the desk, it's editorialcontrol.
- If it sounds like either a SaaS blog or a personal Medium post, it's neither, and it needs rewriting.
