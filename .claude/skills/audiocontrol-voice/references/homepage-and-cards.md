# Reference: homepage, cards, nav

Actual copy from audiocontrol.org. Read these when writing short-form for the site (homepage sections, hardware cards, editor intros, nav, tags).

## Homepage hero

> # Free web editors for the rack samplers you already own.
>
> The hardware is on the shelf. The original software is thirty years
> gone. We write modern browser editors for the 12- and 16-bit samplers
> that built late-80s and 90s records — open source, no install, and
> coming back online one device at a time.
>
> [→ Open S-330 Editor]   [↳ Browse Hardware]

Notes on the shape:
- Headline is a plain-English promise, no hype words.
- Three short sentences in the dek, each carrying a concrete claim.
- The rhythm is: hardware location → software situation → what we do → four tags at the end.
- Arrow link styles: `→` for primary, `↳` for secondary.

## Status-block micro-spec

Homepage uses short status pairs above the fold:

> Available: 02
> In development: 03
> Stack: Open source · Web-native
> Install: None

Zero-padded numbers. Single-word labels. "None" is a complete sentence here.

## Section headers on the homepage

Sentence case, short, declarative. No "Welcome to" openings.

- "Available Now"
- "In Development"
- "Latest Writing"
- "Community Channels"

## Microcopy under section headers

One line, italic-adjacent in tone. Sets context without explaining.

- "Web editors you can open in a browser today."
- "Devices on the workbench. Sign up via GitHub Discussions for updates."
- "Open source. Talk to us. File issues."

## Footer identity line

> Open source web tools for vintage Roland, Akai, and Yamaha samplers and synthesizers.

Single sentence. Enumerates the brands. Doesn't say "passionate" or "community-driven."

## Colophon

> Set in *Departure Mono*, IBM Plex Sans, and JetBrains Mono.
> BUILD 2026.04

Typography credit. Build number. That's it.

## Tag labels

Low-case or Title Case? The site uses Title Case for proper category names ("Available," "Pending," "Hardware," "Docs," "Blog") and lowercase for inline tags. When in doubt, Title Case for the navigational category, lowercase for the inline descriptor.

## Links and CTAs

The site almost never says "Click here" or "Learn more." Link copy is always specific:

- `→ Open Editor`
- `↳ Browse Hardware`
- `→ In development`
- `Read article →`
- `All posts →`
- `→ Editor: [url]`
- `→ Source: [url]`

The arrow stays on the same side (leading `→`, or trailing `→` after a noun phrase). Don't mix them within a single block.
